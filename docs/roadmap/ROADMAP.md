# 路线图

## P0：需求与研究基线

- 建立独立工程目录。
- 完成产品需求、技术设计和数据契约。
- 记录 RunTracker、Brotato 与 ModLoader 的研究结论。
- 确认第三方代码许可证和可复用边界。

## P1：游戏内垂直切片 MVP

- 定义 `CoachSnapshot` 和 `CoachReport`。
- 从当前实时记录和原生运行存档生成统一快照。
- 实现商店购买建议。
- 实现属性缺陷诊断。
- 实现未来 3 波和未来 5 波规划。
- 建立两个黄金样本的回归入口。
- 构建本地 `BrotatoCoach` Mod ZIP。
- 在商店、暂停菜单和结算界面注入离线分析按钮。
- 三个入口复用可滚动、可关闭、支持手柄焦点的结果面板。
- 外部 CLI 仅作为开发和回归工具，不作为用户闭环。

## P2：结算复盘与记录

- 记录波次级属性、经济、武器和受伤趋势。
- 实现成长断档检测。
- 实现失败因果链和下一局改进建议。
- 使用多局真实记录校准阈值。

## P3：轻量采集与兼容性

- 只采集教练所需的最小字段集。
- 不启用网络上传。
- 波次边界写入当前快照，结算时写入完整跑局。
- 避免敌人基类扩展和高频逐击事件。
- 与常用 UI、自动战斗和内容模组完成兼容性测试。

## P4：游戏内体验完善

- 完善结果面板、数据新鲜度和规则版本。
- 改进异常提示、可访问性和本地化文本。
- 完成键盘、鼠标和手柄操作。

## P5：规则库与质量

- 支持原版角色。
- 支持已验证的常用模组角色。
- 建立物品、武器、套装和角色规则版本。
- 建立性能、回归和兼容性测试矩阵。

## P6：GitHub 开源协作

- 发布公开源码仓库和 MVP 预发布包。
- 启用 Issues 和 Pull Requests 接收规则、样本和兼容性贡献。
- 用 CI 自动构建 Mod ZIP 并运行黄金样本验收。
- 建立变更日志、贡献指南和安全报告流程。

## P7：Steam Workshop

- 完成第三方许可证审计。
- 移除不能再分发的上游代码和资源。
- 添加隐私说明、依赖说明和版本兼容说明。
- 小范围测试后发布公测版。
