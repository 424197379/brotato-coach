# ADR-0002：本地日志使用明文 JSON/JSONL

状态：已接受
日期：2026-07-24

## 背景

记录器采集的是游戏内属性、物品、武器、商店和波次聚合值。工程需要让离线规则引擎、
测试工具和人工排查都能直接读取数据，避免额外的解密、专用数据库或二进制格式。

## 决策

- 正式记录器使用 UTF-8 明文 JSON/JSONL。
- 原始 RunTracker、Brotato 和 Mod 样本允许以明文保留。
- 不把 Steam ID、玩家名和本机路径定义为正式教练数据；规则不得依赖这些字段。
- 日志默认只保存在本机，不提供网络上传。
- 对外发布时，黄金样本可以保留游戏字段，但不主动采集与分析无关的平台身份字段。

## 后果

- 日志可被文本工具、Codex 和测试脚本直接读取。
- 单行 JSONL 便于在崩溃后恢复。
- 数据格式不是保密边界；若未来加入云同步，需要另行设计用户授权、传输和删除策略。
