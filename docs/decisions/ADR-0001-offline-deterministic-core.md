# ADR-0001：离线确定性规则核心

状态：已接受
日期：2026-07-24

## 背景

目标体验是在游戏内点击按钮后立即获得教练建议。直接在 Mod 中集成在线
大模型会引入网络、密钥、延迟、费用、隐私和 Workshop 分发问题，也无法
保证同一局面得到稳定结论。

## 决策

首版使用纯离线、确定性、可解释的规则核心：

```text
CoachSnapshot
  -> CharacterProfile
  -> BuildClassifier
  -> ShopAdvisor
  -> StatGapAnalyzer
  -> HorizonPlanner
  -> RunReviewer
  -> CoachReport
```

核心只接收结构化输入并返回结构化报告，不读取游戏节点、不写存档、不
绘制 UI。游戏适配器和外部工具共同调用同一套规则。

## 理由

- 可在 Steam Workshop 中直接分发。
- 无网络、密钥和账号依赖。
- 性能和失败模式可控。
- 可通过固定样本进行回归测试。
- 每条建议可以展示触发规则和证据。
- 未来仍可在游戏外增加可选 AI 解释层。

## 后果

- 需要持续维护角色配置、波次目标和物品知识库。
- 对未知模组角色只能使用通用规则并降低置信度。
- 文案自然度弱于大模型，需要模板和本地化层。
- 所有阈值都必须经实战样本验证，不能凭经验永久写死。

## 明确不做

- 不在核心中发起 HTTP 请求。
- 不在核心中访问 Steam 身份。
- 不让规则核心直接依赖 ModLoader 或 Godot 场景节点。
