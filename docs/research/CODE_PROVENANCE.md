# 代码来源与许可边界

日期：2026-07-24

## 1. 原则

土豆教练源代码采用独立实现。第三方项目可以提供行为、接口和数据结构研究依据，但只有在
许可证明确允许、保留必要声明并完成审查后，代码才能进入仓库或 Workshop 发布包。

## 2. 来源表

| 来源 | 用途 | 当前许可状态 | 可进入发布包 |
|---|---|---|---|
| Brotato 游戏 PCK/脚本 | 确认运行时接口和场景节点 | 商业游戏资源，不再分发 | 否 |
| Brotato ModLoader | 脚本扩展接口 | 发布前复核具体版本许可证 | 仅按许可证使用 |
| RunTracker 1.2.3 | 波次记录行为研究、开发期原始数据 | 当前包未发现 LICENSE | 否 |
| Godot | 引擎与本地工具 | MIT，发布时保留要求 | 按许可证 |
| 土豆教练黄金样本 | 回归测试 | 用户本地游戏数据 | 是 |
| 土豆教练原创实现 | Mod、记录器、规则和工具 | 待项目确定许可证 | 是 |

## 3. RunTracker clean-room 边界

允许记录：

- 哪些生命周期会发生。
- 输入字段和输出 JSON 中可观察的数据。
- 性能与兼容性问题。
- 教练需要但 RunTracker 未提供的字段。

不得复制：

- GDScript 函数体。
- 类、文件和私有协议的整体组织。
- 上传器、队列和账号逻辑。
- 未获授权的文本、图标和其他资源。

开发实现由另一任务依据 `TECHNICAL_DESIGN.md` 和 `DATA_CONTRACT.md` 编写，不以第三方源码
作为复制模板。代码评审需要检查来源说明和相似实现风险。

## 4. 游戏资源边界

- 仓库不保存反编译游戏脚本和场景。
- Workshop 包不包含 Brotato 原始素材。
- 运行时通过游戏已有资源 ID 和节点接口读取数据。
- 测试夹具可以保存玩家自己的运行 JSON，但不能打包完整游戏数据库。

## 5. 发布前检查

- 为仓库选择并添加项目许可证。
- 复核 ModLoader 与所有构建工具许可证。
- 搜索第三方版权头、Workshop ID 和复制代码片段。
- 确认 `third_party/NOTICE.md` 与最终依赖一致。
- 从干净检出构建发布包，证明不依赖本机第三方源码目录。
